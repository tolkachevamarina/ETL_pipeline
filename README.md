#  API Integration & ETL Pipeline

## 📌 Опис проєкту

Основою проєкту є база даних **ClassicModels**, яка містить інформацію про клієнтів, замовлення, товари, співробітників та платежі. Додатково за допомогою API було отримано курси валют та сформовано окрему таблицю **currency_rates**.

Проєкт демонструє навички роботи з API, базами даних, обробкою даних та побудовою ETL-процесів за допомогою Python.

---

## 🗄️ Структура бази даних

У проєкті використовувалась база даних **ClassicModels**, яка була доповнена таблицею `currency_rates`, створеною на основі даних, отриманих через API.

| Таблиця        | Опис                            |
| -------------- | ------------------------------- |
| customers      | Інформація про клієнтів         |
| employees      | Дані про співробітників         |
| offices        | Інформація про офіси компанії   |
| orderdetails   | Деталі замовлень                |
| orders         | Інформація про замовлення       |
| payments       | Дані про платежі клієнтів       |
| productlines   | Категорії товарів               |
| products       | Інформація про товари           |
| currency_rates | Курси валют, отримані через API |

---

## 🔌 Інтеграція з API та збереження в БД

На першому етапі проєкту було реалізовано інтеграцію із зовнішнім API для отримання актуальних курсів валют.

Отримані дані були:

* завантажені через HTTP-запит;
* оброблені та приведені до табличного формату;
* збережені в базу даних MySQL у новостворену таблицю currency_rates.

Таким чином база даних ClassicModels була доповнена інформацією про валютні курси, що дозволяє використовувати її для подальших аналітичних задач та фінансових розрахунків.

## 🔄 Етапи ETL-процесу

### 1. Extract

На цьому етапі витягували для подальшої обробки наступні дані:
* виконані замовлення за 2004 рік - для цього об'єднали таблиці orders, orderdetails, products та customers;
* продукти - назви, категорії, ціни;
* курси валют.

### 2. Transform

1. Було додано розрахункові колонки до основної таблиці:
* `profit_per_item` - прибуток з одного товару;
* `total_profit` - загальний прибуток з товарної позиції
* `total_amount_eur` - сума в євро.

 2. Створено аналітичну таблицю "ТОП-5 найприбутковіших країн".

    Було розраховано наступні метрики по кожній країні:
* Кількість унікальних замовлень;
* Загальний дохід;
* Загальний прибуток;
* Кількість проданих товарів;
* Маржа прибутку (%).

 3. Створено аналітичну таблицю по продуктових лініях.
 
     Було розраховано наступні метрики по кожній продуктовій лінії:
* Кількість унікальних замовлень;
* Загальний дохід;
* Загальний прибуток;
* Кількість проданих товарів;
* Маржа прибутку (%). 

 4. Було розраховано загальні показники бізнесу за 2004 рік
    
     Фінансові показники:
* Загальний дохід в доларах;
* Загальний дохід в євро;
* Загальний прибуток в доларах;
* Загальна маржа прибутку (%);
* Середній розмір замовлення.
   
     Операційні показники:
* Кількість унікальних замовлень;
* Кількість унікальних клієнтів;
* Період даних.

     Топ показники:
* Найприбутковіша країна;
* Найприбутковіша продуктова лінія.
     
### 3. Load

Зберігаємо результати обчислень:

1. Створено Excel файл з трьома вкладками:

* "Summary" - підсумкова інформація у вигляді таблиці "Показник - Значення";
* "Top_Countries" - аналітика по ТОП-5 країнах;
* "Product_Lines" - аналітика по всіх продуктових лініях.

2. Створено 2 візуалізації:

* стовпчикова діаграма топ-5 країн за доходом;
* Створено pie chart з відсотковим розподілом доходу в USD по продуктових лінійках.
